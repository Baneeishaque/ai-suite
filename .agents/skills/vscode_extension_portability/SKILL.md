# VS Code Extension Link Portability Skill

## 1. Scope Statement

This skill establishes the industrial protocol for identifying and refactoring non-portable or machine-specific file paths generated by VS Code extensions within `settings.json` profiles. It mandates the detection of absolute home prefixes (`/Users/dk/`) and ephemeral cache locations (`/var/folders/`), replacing them with portable tilde (`~`) expansion and permanent extension storage mappings. The protocol prioritizes configuration portability across environment migrations and ensures long-term reference stability.

***

## 3. Protocol Layers

The protocol is organized into five operational phases.

### 3.1 Phase 1: Detection & Audit

1.  **Scanning**: Inspect `settings.json` for "red flag" patterns:
    - `/var/folders/` or `/tmp/` (Temporary Cache)
    - `/Users/` or `C:\Users\` (Absolute Home Prefixes)
    - Ephemeral extension storage paths.
2.  **Inventory**: Present a table of non-portable links discovered, categorizing them by risk (Temporary vs. Absolute).

### 3.2 Phase 2: Refactor Strategy

1.  **Home Expansion**: Map absolute home prefixes to the tilde (`~`) character.
2.  **Permanent Mapping**: Identify the extension's stable internal directory (e.g., `~/.antigravity/extensions/<extension-id>/`) for items currently in cache.
3.  **Atomic Backup**: Create a `.bak` copy of `settings.json` before any automated replacement.

### 3.3 Phase 3: Automated Refactoring

1.  **Script Execution**: Utilize the `refactor_links.py` script for precise, regex-based path substitution.
2.  **Dry Run Verification**: Run with `--dry-run` to audit proposed changes without disk modification.

### 3.4 Phase 4: Verification & Validation

1.  **Syntax Audit**: Ensure the `settings.json` remains a valid JSON document using `jq .`.
2.  **Link Fidelity**: Verify that the refactored paths logically point to the intended permanent resources.

### 3.5 Phase 5: Synchronization

1.  **VCS Update**: Update the configuration source-of-truth repository (e.g., `configurations-private`) to ensure the portable links are persisted globally.
2.  **Atomic Commit**: Commit the changes using the [Git Atomic Commit Skill](../git_atomic_commit/SKILL.md).

***

## 4. Deep Command Explanation

### 4.1 `python3 refactor_links.py --file settings.json`
- Executes regex-based path substitution to convert absolute or temporary paths into portable formats.
- Supports tilde expansion and extension-specific permanent directory mapping.

### 4.2 `grep -r "/var/folders/" settings.json`
- Performs a recursive search for temporary OS cache strings.
- Critical for identifying ephemeral links that will break after a system reboot or migration.

### 4.3 `jq . settings.json`
- Validates that the JSON structure was not corrupted during regex-based refactoring.
- A mandatory exit-gate for any configuration modification protocol.

***

## 5. Prohibited Behaviors

- **No absolute paths**: Never leave `/Users/` or `C:\Users\` prefixes in a shared configuration file.
- **No ephemeral links**: All `/var/folders/` links MUST be refactored to permanent locations.
- **No skipping backups**: A `.bak` file is mandatory before modification.
- **No manual replacement of complex objects**: Use the automation script to ensure regex precision and prevent syntax errors.
- **No assuming portability**: Always verify the tilde-expanded path works in the target environment.
